**Key pattern — the conversion replaces:**
```go
// BEFORE (raw SQL):
{
    ID: "2024_01_15_000002_create_books",
    Migrate: func(db *gorm.DB) error {
        return db.Exec(`CREATE TABLE IF NOT EXISTS "books" (...) `).Error
    },
}

// AFTER (AutoMigrate):
{
    ID: "2024_01_15_000002_create_books",
    Migrate: func(db *gorm.DB) error {
        return db.AutoMigrate(&models.Book{})
    },
}
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Migration runner → database | Migration code executes DDL against PostgreSQL database |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | migration IDs | mitigate | Do NOT change IDs — tracked in database. Changing an ID causes re-execution on existing databases, which would attempt to re-create existing tables. |
| T-04-02 | Information Disclosure | trace logs | accept | Migration DDL may leak schema details in logs. Acceptable — no PII in table definitions. |
| T-04-03 | Elevation of Privilege | AutoMigrate scope | mitigate | AutoMigrate creates tables with permissions of the database user. Ensure the user has only schema-level permissions (not superuser). This is already handled by the ReusableMigrator pattern. |
| T-04-04 | Repudiation | schema mismatch | mitigate | GORM AutoMigrate is declarative — the schema is DEFINIED by model structs. By replacing raw SQL, we ensure schema ALWAYS matches code. Model struct IS the source of truth. |
</threat_model>
