    func (r *RepositoryName) FindByResourceID(ctx context.Context, rid *helpers.ResourceID) (*models.ModelType, error) {
        var record models.ModelType
        query := r.db.WithContext(ctx)
        if rid.UUID != guuid.Nil {

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→API | Untrusted ID values (UUID strings or uint strings) cross here via route params, query params, and request bodies |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | S (Spoofing) | ParseParamID / ParseQueryID | mitigate | Validate UUID strings using google/uuid.Parse before accepting; validate uint strings with strconv.ParseUint. Reject malformed input with appErr.NewBadRequestError (D-05) |
| T-05-02 | T (Tampering) | DTO dual fields | mitigate | UUID fields are optional (*uuid.UUID pointer); existing uint validation via binding:"required" still applies. If both uint and UUID are set, service checks UUID first (D-07). No concurrent modification risk (single request context) |
| T-05-03 | I (Info Disclosure) | Repository FindByResourceID | accept | WHERE id = ? OR uuid = ? exposes whether either ID exists via different error messages. Mitigated by GORM's gorm.ErrRecordNotFound which returns identical error whether ID exists or not. Same behavior as existing FindByID |
| T-05-04 | D (DoS) | ParseParamID / ParseQueryID | mitigate | UUID parsing and uint parsing are O(1) bounded operations. No risk of resource exhaustion from input length (uuid is fixed 36 chars, uint parsed up to 20 digits) |
</threat_model>
