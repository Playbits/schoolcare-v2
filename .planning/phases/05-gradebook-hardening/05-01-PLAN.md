# Plan 05-01: Integer Basis-Point Score Migration

## Objective
Complete the dual-phase migration of score-related columns from float32 to integer basis points (BP). The models already have `*BasisPoints` columns alongside legacy float columns (`MaxScoreBasisPoints`, `TotalBasisPoints`, `MinBasisPoints`, `MaxBasisPoints`). This plan ensures all computation reads from and writes to the BP columns, adds any missing BP columns, migrates existing Score JSONB data, adds a migration to populate BP columns from float values for existing tenants, and drops the legacy float columns via a future migration.

## Requirements
PREC-01, PREC-02

## Success Criteria
- [ ] All grade item max scores stored as int basis points in `MaxScoreBasisPoints` (write on create/update)
- [ ] All assessment totals stored as int basis points in `TotalBasisPoints` (write on create/update)
- [ ] All grade boundary min/max values stored as int basis points (write on create/update)
- [ ] `Score.Score` JSONB stores per-grade-item scores as int basis points (already done in `buildGradeItemScoreJSON` and `mergeGradeItemScore` — verify and backfill)
- [ ] All computation paths (`computeGradeItemsTotal`, `CalculateGrade`, `CalculateGradeForAssessmentTotal`, rollup) use int BP exclusively — no float32 arithmetic for boundary comparisons
- [ ] New migration populates existing `*BasisPoints` columns from float values for all tenant schemas
- [ ] `go build ./...`, `go vet ./...`, tests pass

## Tasks

### 1. Audit and fix computation paths to use int basis points exclusively

- [ ] **1.1** Search every service file for float32 computations involving scores, max scores, or totals. Replace `Score.Total`, `GradeItem.MaxScore`, `GradeBoundary.MinScore`/`MaxScore` arithmetic with their `*BasisPoints` counterparts.
  - `backend/internal/modules/grading/service.go` — `CalculateGrade`: already uses BP via `RoundToBasisPoints` and `percentageBP`, verify `entryMinBP`/`entryMaxBP` helpers
  - `backend/internal/modules/grading/service.go` — `CalculateGradeForAssessmentTotal`: same verification
  - `backend/internal/modules/score/service.go` — `computeGradeItemsTotal`: already uses `MaxScoreBasisPoints` and int BP
  - `backend/internal/modules/score/service.go` — `RollupAssessmentScore`: verify `assessment.TotalBasisPoints` is used
  - `backend/internal/modules/score/service.go` — `BulkSaveScores`: already uses `MaxScoreBasisPoints` from `giMap`
  - `backend/internal/modules/academic/` — search for `assessment.Total` or `gradeItem.MaxScore` used in FP arithmetic; replace with `TotalBasisPoints`/`MaxScoreBasisPoints`
- [ ] **1.2** Create/update DTOs: ensure `CreateGradeItemRequest`, `UpdateGradeItemRequest`, `CreateAssessmentRequest`, `UpdateAssessmentRequest`, `CreateGradeBoundaryRequest`, `UpdateGradeBoundaryRequest` accept BasisPoints fields and auto-compute from float values via `RoundToBasisPoints`.
  - Existing `*BasisPoints` fields on models may need `gorm:"-"` or explicit migration to be writable.

### 2. Migration to backfill existing `*BasisPoints` columns

- [ ] **2.1** Add a new migration file at `backend/internal/database/migrations/school/school.go` (new entry in `SchoolMigrations()`):
  - Existing models already have `MaxScoreBasisPoints`, `TotalBasisPoints`, `MinBasisPoints`, `MaxBasisPoints` columns — ensure they exist via `ALTER TABLE ADD COLUMN IF NOT EXISTS`
  - Backfill: `UPDATE grade_items SET max_score_basis_points = ROUND(COALESCE(max_score, 0) * 100) WHERE max_score_basis_points = 0 OR max_score_basis_points IS NULL`
  - Same for: `assessments.total_basis_points`, `grade_boundaries.min_basis_points`, `grade_boundaries.max_basis_points`
  - Backfill existing Score JSONB blobs: parse each row, convert any float scores to int BP, re-serialize.

### 3. Add write-side synchronization for BasisPoints columns

- [ ] **3.1** In `academic/service.go` `CreateGradeItem`/`UpdateGradeItem`: ensure `MaxScoreBasisPoints` is set from `MaxScore` using `RoundToBasisPoints` when creating/updating.
- [ ] **3.2** In `academic/service.go` `CreateAssessment`/`UpdateAssessment`: ensure `TotalBasisPoints` is set from `Total`.
- [ ] **3.3** In `grading/service.go` `CreateGradeBoundary`/`UpdateGradeBoundary`: set `MinBasisPoints`/`MaxBasisPoints` from `MinScore`/`MaxScore`.
- [ ] **3.4** Verify `backend/internal/modules/score/service.go` `SaveGradeItemScore` already stores scores as BP (via `RoundToBasisPoints` call at line 132). Confirm `mergeGradeItemScore` and `buildGradeItemScoreJSON` both store int values.
