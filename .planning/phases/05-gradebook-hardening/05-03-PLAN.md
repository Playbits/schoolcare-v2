# Plan 05-03: Grade Freeze on Session Completion

## Objective
Implement the complete grade freeze experience. The backend trigger and `canModifyScore` helper already exist. This plan adds the `FrozenBanner` UI component, verifies all toolbar buttons pass the `disabled` prop correctly, adds the `RecalcWarningBadge` component, and ensures the frontend state machine (normal ‚Üí frozen ‚Üí override-active ‚Üí re-frozen) renders correctly per the UI-SPEC.

## Requirements
PREC-05

## Success Criteria
- [ ] `FrozenBanner` component renders amber/warning full-width banner with Lock icon when session is completed
- [ ] `FrozenBanner` renders green/info banner with countdown when grade override is active
- [ ] `FrozenBanner` renders null when neither frozen nor overridden
- [ ] All ScoreGrid toolbar buttons (`Download Excel`, `Import Excel`, `Rollup All`, `Save All`) are disabled when `disabled=true` ‚Äî verify `disabled` prop is passed to each
- [ ] Score input cells show `opacity-60 cursor-not-allowed` styling and `aria-label`/`title` attributes when disabled
- [ ] Score cell comment popover button disabled when frozen
- [ ] Session cards show Lock icon + "Frozen" badge when `session.status === "completed"` (already done ‚Äî verify)
- [ ] "Override Active" badge + countdown timer visible when override active (already done ‚Äî verify)
- [ ] `RecalcWarningBadge` component renders on assessment cards with `recalc_needed === true`
- [ ] Frontend builds (`npx tsc --noEmit` passes, `yarn build` passes)
- [ ] No regressions in existing teacher academics page

## Tasks

### 1. Audit existing grade freeze implementation

- [ ] **1.1** Verify the existing DB trigger (`trg_check_grade_freeze`) works correctly:
  - Function `check_grade_freeze()` checks `session.status = 'completed'` and `grade_override IS DISTINCT FROM true`
  - Trigger is `BEFORE INSERT OR UPDATE ON scores`
  - Confirm it fires and blocks INSERT/UPDATE when session is completed and no override
- [ ] **1.2** Verify `canModifyScore` in `score/service.go` already:
  - Returns error when `session.Status == "completed"` and no active override
  - Bypasses when `GradeOverride == true` and not expired
- [ ] **1.3** Verify existing frontend freeze indicators in `teacher.academics.tsx`:
  - Session card shows Lock icon + Frozen badge for completed sessions
  - Override Active badge + countdown works
  - `ScoreGrid` receives `disabled` prop passed through `ScoreGridWrapper`

### 2. Create FrozenBanner component

- [ ] **2.1** Create `frontend/src/components/academics/frozen-banner.tsx`:
  - Props: `isFrozen: boolean`, `isOverridden: boolean`, `overrideExpiresAt?: string`
  - **Frozen state** (`isFrozen && !isOverridden`):
    - Full-width amber/warning banner with `bg-amber-50 border-amber-200` styling
    - `Lock` icon with `aria-label="Grades frozen"`
    - Heading: "Grades Frozen ‚Äî Session Completed"
    - Body: "This session's grades are locked. Contact an administrator to enable corrections."
  - **Override active** (`isOverridden`):
    - Full-width green/info banner with `bg-green-50 border-green-200` styling
    - `ShieldAlert` icon
    - Heading: "Grade Override Active"
    - Body: "Scores can be edited for {time remaining}. All changes are logged to the correction audit trail."
    - Countdown: `<time datetime={overrideExpiresAt}>Expires {formattedDate}</time>`
  - **Neither**: renders `null`
- [ ] **2.2** Integrate `FrozenBanner` into `score-grid.tsx` above the toolbar, passing props based on session status.

### 3. Verify toolbar button disabled state

- [ ] **3.1** In `frontend/src/components/academics/score-grid.tsx`:
  - Read the existing `disabled` prop and verify it passes to ALL toolbar buttons:
    - "Download Excel" button ‚Üí `disabled={disabled}`
    - "Import Excel" button ‚Üí `disabled={disabled}`
    - "Rollup All" button ‚Üí `disabled={disabled}`
    - "Save All" button ‚Üí `disabled={disabled}`
  - Add `title` attribute to each disabled button: `title="Session completed ‚Äî scores are frozen"`
  - Add `aria-disabled="true"` when disabled
- [ ] **3.2** Disable comment popover button when `disabled={true}` ‚Äî add `disabled` prop to the comment trigger button.

### 4. Create and integrate RecalcWarningBadge

- [ ] **4.1** Create `frontend/src/components/academics/recalc-warning-badge.tsx`:
  - Props: `recalcNeeded: boolean`
  - Amber `Badge` with "Recalculation Needed" text
  - Tooltip: "Grade items have changed. Run rollup to recalculate assessment totals."
  - Renders `null` when `recalcNeeded === false`
- [ ] **4.2** In `teacher.academics.tsx`, show `RecalcWarningBadge` on each assessment card where `assessment.recalc_needed === true`.
