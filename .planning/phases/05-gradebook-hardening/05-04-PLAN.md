# Plan 05-04: Versioned Grade Corrections (Admin Override, Audit Trail)

## Objective
Build the complete versioned grade corrections system. Admin override already has a working endpoint (`EnableGradeOverride`) and frontend trigger. This plan adds the `grade_corrections` database table, backend endpoints for recording corrections and fetching correction history, and the frontend `CorrectionHistoryPanel`, `OverrideReasonDialog`, and correction reason flow during score save.

## Requirements
PREC-07

## Success Criteria
- [ ] `grade_corrections` model/table created with columns: `score_id`, `admin_id`, `before` (JSONB), `after` (JSONB), `reason` (text), `created_at`
- [ ] Composite indexes on `(score_id, created_at)` and `(admin_id, created_at)`
- [ ] Append-only — no UPDATE/DELETE allowed via GORM (use `Create` only)
- [ ] `POST /academic/scores/bulk` extended to accept optional `correction_reason` field — when provided, records a `GradeCorrection` for each mutated score
- [ ] `GET /academic/scores/:assessmentId/corrections` endpoint returns correction history (including resolved `admin_name` via `UserInfo`)
- [ ] Frontend `OverrideReasonDialog` — enhanced override confirmation with reason textarea
- [ ] Frontend `CorrectionHistoryPanel` — Sheet-based DataTable showing before/after diff
- [ ] "View Corrections" button in ScoreGrid toolbar when corrections exist
- [ ] `GradeCorrection` TypeScript interface + `useCorrectionHistory` hook in `useAcademics.ts`
- [ ] `go build ./...`, `go vet ./...`, frontend `yarn build` pass

### 1. Create GradeCorrection model and migration

- [ ] **1.1** Add `GradeCorrection` model to `backend/internal/database/models/` (new file `grade_correction.go` or add to existing score-related models):
  ```go
  type GradeCorrection struct {
      ID        uint           `gorm:"primaryKey" json:"id"`
      ScoreID   uint           `gorm:"index;not null" json:"score_id"`
      AdminID   uint           `gorm:"index;not null" json:"admin_id"`
      Before    datatypes.JSON `gorm:"type:jsonb;not null" json:"before"`
      After     datatypes.JSON `gorm:"type:jsonb;not null" json:"after"`
      Reason    string         `gorm:"type:text;not null" json:"reason"`
      CreatedAt time.Time      `json:"created_at"`

      Score *Score `gorm:"foreignKey:ScoreID" json:"-"`
  }
  ```
- [ ] **1.2** Add migration entry in `school.go`:

### 6. Frontend: CorrectionHistoryPanel + hooks

- [ ] **6.1** Add `GradeCorrection` interface and hooks in `useAcademics.ts`:
  ```typescript
  export interface GradeCorrection {
    id: number;
    score_id: number;
    admin_id: number;
    admin_name?: string;
    before: Record<string, number>;
    after: Record<string, number>;
    reason: string;
    created_at: string;
  }

  export function useCorrectionHistory(assessmentId: number) {
    return useQuery<GradeCorrection[]>({
      queryKey: [...queryKeys.gradeItemScores, "corrections", assessmentId],
      queryFn: () => api.get(`/academic/scores/${assessmentId}/corrections`),
      enabled: !!assessmentId,
    });
  }
  ```
  - Extend `useBulkSaveScores` to accept optional `correction_reason` parameter.
- [ ] **6.2** Create `frontend/src/components/academics/correction-history-panel.tsx`:

  - Before/After diff cells use `<del>` and `<ins>` semantic HTML elements for screen readers
  - Error state: toast "Failed to load correction history"
