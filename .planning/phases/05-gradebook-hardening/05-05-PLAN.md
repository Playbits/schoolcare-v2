# Plan 05-05: Boundary Test Suite

## Objective
Augment the existing WAEC grade boundary tests with comprehensive coverage for all Phase 5 features: basis-point precision at every WAEC threshold (A1–F9), sum-to-100 trigger validation, grade freeze enforcement, grade correction recording/retrieval, and report card snapshotting. All tests must pass at `±0.01` precision (1 basis point).

## Requirements
PREC-01, PREC-02, PREC-03, PREC-04, PREC-05, PREC-06, PREC-07, PREC-08

## Success Criteria
- [ ] All 8 WAEC thresholds (A1, B2, B3, C4, C5, C6, D7, E8 → F9) tested at exact boundary, -0.01, and +0.01 precision using integer basis points
- [ ] Sum-to-100 trigger tested: valid insert passes, overshoot insert fails with clear error
- [ ] Grade freeze tested: INSERT/UPDATE on scores blocked for completed session, allowed with active override, blocked after override expires
- [ ] Grade correction recorded and retrieved via API — verify before/after snapshots and reason stored correctly
- [ ] Report card snapshot stores values not references (smoke test: verify snapshot differs from live after score change)
- [ ] `go test ./... -count=1` passes for affected packages (grading, score, academic, mathutil)

## Tasks

### 1. Augment existing WAEC boundary tests for full precision coverage

- [ ] **1.1** Add test functions in `backend/internal/modules/grading/service_test.go` covering ALL 8 WAEC boundaries at ±0.01:
  - Create a table-driven test `TestCalculateGrade_AllWAECBoundaries` that iterates:
    - A1: min=7500 BP (75%), max=10000 BP (100%) — test 74.99 (B2), 75.00 (A1), 75.01 (A1)
    - B2: min=7000 BP (70%), max=7499 BP (74.99%) — test 69.99 (B3), 70.00 (B2), 70.01 (B2)
    - B3: min=6500 BP (65%), max=6999 BP (69.99%) — test 64.99 (C4), 65.00 (B3), 65.01 (B3)
    - C4: min=6000 BP (60%), max=6499 BP (64.99%) — test 59.99 (C5), 60.00 (C4), 60.01 (C4)
    - C5: min=5500 BP (55%), max=5999 BP (59.99%) — test 54.99 (C6), 55.00 (C5), 55.01 (C5)
    - C6: min=5000 BP (50%), max=5499 BP (54.99%) — test 49.99 (D7), 50.00 (C6), 50.01 (C6)
    - D7: min=4500 BP (45%), max=4999 BP (49.99%) — test 44.99 (E8), 45.00 (D7), 45.01 (D7)
    - E8: min=4000 BP (40%), max=4499 BP (44.99%) — test 39.99 (F9), 40.00 (E8), 40.01 (E8)
    - F9: min=0 BP (0%), max=3999 BP (39.99%) — test 0 (F9), 39.99 (F9)
  - Each case verifies `RoundToBasisPoints` output matches expected BP value
  - Each case verifies grade letter via `CalculateGrade` and `CalculateGradeForAssessmentTotal`
- [ ] **1.2** Add edge-case tests:
  - Score > 100% → capped at 100%
  - Score < 0% → treated as 0%
  - Empty or nil grade scale → returns "-" grade
  - Max score 0 → no division by zero

### 2. Add sum-to-100 trigger tests

- [ ] **2.1** Create `backend/internal/modules/academic/sum_trigger_test.go`:
  - `TestSumTo100Trigger_ValidInsert`: insert grade items whose `max_score_basis_points` sum to `assessment.total_basis_points` — expect success
  - `TestSumTo100Trigger_OverflowInsert`: insert grade items whose sum exceeds total — expect error with message containing "Grade items sum BP"
  - `TestSumTo100Trigger_Update`: update a grade item's max_score to cause overflow — expect error
  - `TestSumTo100Trigger_DeleteRemovesExcess`: insert overshooting, delete one item to bring sum within total — expect success
  - Use `gorm.io/driver/sqlite` for in-memory test DB (matching existing test pattern) or raw PostgreSQL via testcontainers

### 3. Add grade freeze enforcement tests

- [ ] **3.1** Create `backend/internal/modules/score/freeze_test.go`:
  - `TestGradeFreeze_BlocksInsertOnCompletedSession`: create session with `status = "completed"`, attempt to insert score — expect error from DB trigger
  - `TestGradeFreeze_BlocksUpdateOnCompletedSession`: create score on active session, complete session, attempt to update score — expect error from `canModifyScore`
  - `TestGradeFreeze_AllowsWithActiveOverride`: set `grade_override = true` and `grade_override_expires_at = future`, attempt insert/update — expect success
  - `TestGradeFreeze_BlocksAfterOverrideExpired`: set override with past expiry, attempt insert — expect error
  - `TestGradeFreeze_AllowsOnActiveSession`: normal flow — expect success
  - Test both the app-level `canModifyScore` helper AND the DB trigger

### 4. Add grade correction recording/retrieval tests

- [ ] **4.1** Create `backend/internal/modules/score/correction_test.go`:
  - `TestGradeCorrection_RecordedOnSave`: save a score with `correction_reason`, verify `GradeCorrection` record exists with correct `before`/`after` snapshots
  - `TestGradeCorrection_RetrievedByAssessment`: create multiple corrections, fetch by assessment ID, verify all returned
  - `TestGradeCorrection_AdminNameResolved`: verify admin name is resolved from UserInfo in response
  - `TestGradeCorrection_AppendOnly`: attempt to update/delete a correction — expect error (no UPDATE/DELETE routes)
  - `TestGradeCorrection_NoReasonNoRecord`: save score without `correction_reason` — verify no correction record created

### 5. Add report card snapshot smoke test
