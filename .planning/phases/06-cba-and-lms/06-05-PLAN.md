# 06-05: LMS Quiz Engine (Powered by CBA)

## Objective
Wire the existing `content_type: "quiz"` in LMS lessons to the CBA exam engine. Instead of building a new quiz system, teachers select an existing CBA paper, and students take the quiz inline within the lesson — reusing all CBA infrastructure (questions, auto-grading, results).

## Tasks

### Task 1: Add `cba_paper_id` FK to `Lesson` model
1. Add nullable `cba_paper_id` (uint → `CBAPaper` model) to `Lesson` model
2. Add migration in school migration file
3. Only relevant when `content_type = "quiz"`; ignored for other content types

### Task 2: Lesson editor — quiz paper selector
1. In the lesson editor dialog (from 06-03), when Content Type = "quiz":
   - Show a paper selector dropdown (fetches `GET /cba/papers`)
   - Teacher picks a paper, the lesson saves with `cba_paper_id` set
   - Show paper title, question count, and duration in the dropdown

### Task 3: Lesson detail — quiz rendering
1. On `lms.$courseId.tsx`, when rendering a lesson with `content_type = "quiz"`:
   - Show "Start Quiz" button (for students with active enrollment)
   - On click: call `POST /cba/exams/start` with the CBA assignment linked to the paper
   - If no CBA assignment exists: show "Quiz not yet configured" for teacher, hidden for student
   - Embed the exam-taking experience from `cba.exams.$examId.take.tsx` (reuse component)
   - On submit: show results inline (score, grade, pass/fail) below the lesson content
   - On revisit: show "Completed — Score: X%" with option to view full results

### Task 4: Quiz assignment auto-creation
1. Add a helper that auto-creates a CBA assignment for the paper if one doesn't exist for this lesson:
   - Use lesson title as assignment title
   - Use course's level/subject for level_id/subject_id (after 06-02)
   - Set schedule window to match course active dates
   - Create once, cache the assignment ID on the lesson
