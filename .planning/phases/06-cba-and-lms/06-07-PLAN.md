# 06-07: CBA → Intake Enrollment Exam

## Objective
Enable schools to use CBA exams as entrance/placement exams during the admission intake process. An `AdmissionIntake` can be configured with a CBA paper — applicants take the exam as part of their application, and results are recorded in `EntranceExamResult` for screening.

## Tasks

### Task 1: Add `cba_paper_id` FK to `AdmissionIntake` model
1. Add nullable `cba_paper_id` (uint → `CBAPaper` model) to `AdmissionIntake`
2. Add migration in school migration file
3. Only relevant for intakes that require entrance exams; legacy intakes unaffected

### Task 2: Update AdmissionIntake DTOs and handler
1. Add `cba_paper_id` / `cba_paper_uuid` to `CreateIntakeRequest` and `IntakeResponse` DTOs
2. Update `CreateIntake`/`UpdateIntake` handlers to accept the field
3. Expose resolved CBA paper name in `IntakeResponse`

### Task 3: Applicant CBA exam taking flow
1. When an application's intake has a `cba_paper_id`:
   - Show an "Take Entrance Exam" section on the application portal (public page)
   - After application is submitted (status ≥ "submitted"), applicant can start the exam
2. Start exam: auto-create a one-off `CBAAssignment` + `ExamSession` for the applicant:
   - Title: `"Entrance Exam — {Intake.Name}"`
   - Student = applicant's eventual student record (or a placeholder)
   - Timer, questions, auto-grading from CBA engine
3. Submit exam: record score in `EntranceExamResult`:
   - `ApplicationID` = the application
   - `Subject` = "General" or the intake's target subject
   - `Score` / `TotalMarks` from the CBA session
4. On completion, auto-transition application status to "screening" if not already

### Task 4: Admin — intake exam configuration UI
1. On the intake configuration page, add a "CBA Entrance Exam" section
2. Dropdown to select a CBA paper (fetches `GET /cba/papers`)
3. Show paper title, question count, duration in the selector
4. Clear/reset option to remove the exam link
