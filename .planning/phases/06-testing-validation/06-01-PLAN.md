This confirms the existing TestNonceUniqueness but with stronger assertions (base64 decode and check nonce portions differ). Use `t.Parallel()`.

    2. `TestDecrypt_EmptyCiphertext` — `s.Decrypt("", "aad")` should return error wrapping "decode base64".
       `s.Decrypt("", "")` — empty AAD should work but empty ciphertext should fail.
       Use `t.Parallel()`.

    3. `TestDecrypt_WrongAADEdgeCases` — Table-driven:
       - Encrypt with empty AAD "", decrypt with non-empty "aad" → error
       - Encrypt with "aad", decrypt with empty "" → error  
       - Encrypt with short AAD "ab", decrypt with different short AAD "cd" → error
       Use `assert.ErrorContains(t, err, "AAD mismatch or corrupted data")`.

    4. `TestEncryptDecrypt_WithSpecialCharacters` — Test with:
       - Plaintext containing null bytes: "data\x00with\x00nulls"
       - Plaintext containing newlines: "line1\nline2\nline3"
       - AAD containing special regex characters: `.*+?^$()[]{}|\\`
       Use table-driven, roundtrip verify.

    5. `TestDecrypt_Base64PaddingVariations` — GCM output is nonce+ciphertext which when base64-encoded may have padding. Test:
       - Known valid ciphertext with padding removed (expect decode error)
       - Spaces in base64 string (some systems add spaces — our DecodeString handles this)
       Use `t.Parallel()`.

    6. `TestEncrypt_LargeAAD` — AAD of 64KB: roundtrip should succeed.
       Use `t.Parallel()`.

    7. `TestNewService_KeyEdgeCases` (extend existing validation tests):
       - Key with 63 hex chars (not 64) — expect error about length
       - Key with leading/trailing whitespace — expect "decode encryption key" error since hex decode fails
       Use sub-tests.

    Follow existing test patterns: `testify/assert+require`, `t.Parallel()` at function level, table-driven where applicable.
