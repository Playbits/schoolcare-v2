    ```go
    func TestCrossTenantIsolation_MiddlewareChain(t *testing.T) {
        t.Parallel()
        
        cm := NewConnectionManager(nil, newTestCryptoService(t), nil)
        close(cm.done)
        defer func() { cm.conns.Lock(); cm.clients = make(map[uint]*gorm.DB); cm.conns.Unlock() }()
        
        cm.conns.Lock()
        cm.clients[42] = &gorm.DB{Config: &gorm.Config{}}
        cm.clients[99] = &gorm.DB{Config: &gorm.Config{}}
        cm.conns.Unlock()
        
        // Test that factory correctly routes different schools to different repos
        factory := NewRepositoryFactory(cm, &gorm.DB{})
        
        t.Run("school 42 gets correct repos", func(t *testing.T) {
            repos, err := factory.ForSchool(42)
            require.NoError(t, err)
            assert.Equal(t, uint(42), repos.SchoolID())
            assert.NotNil(t, repos.TenantDB())
        })
        
        t.Run("school 99 gets correct repos", func(t *testing.T) {
            repos, err := factory.ForSchool(99)
            require.NoError(t, err)
            assert.Equal(t, uint(99), repos.SchoolID())
            assert.NotNil(t, repos.TenantDB())
        })
        
        t.Run("school 42 and school 99 have different DB connections", func(t *testing.T) {
            repos42, _ := factory.ForSchool(42)
            repos99, _ := factory.ForSchool(99)
            assert.NotSame(t, repos42.TenantDB(), repos99.TenantDB(),
                "School isolation violation: two schools should never share a DB connection")
        })
    }
    ```

    # SchoolCare v2 — Deployment Guide
    
    ## Architecture Overview
    Shared core PostgreSQL for cross-tenant data (users, schools, auth).
    Per-tenant dedicated PostgreSQL databases for school data (students, scores, etc.).
    Redis for caching (tenant context, rate limiter, blacklist).
    S3-compatible storage for backups.
    
    ## Prerequisites
    - PostgreSQL 16+
    - Redis 7+
    - S3-compatible storage (AWS S3, MinIO, DigitalOcean Spaces)
    - pg_dump 16+, pg_restore 16+
    - ENCRYPTION_KEY: 64 hex chars (32 bytes AES-256-GCM)
    
    ## Provisioning a New School Database
    1. Register school in core DB (via Laravel admin panel): gets school_id (e.g., 42)
    2. Create dedicated PostgreSQL database for the school
    3. Encrypt the database password using the Go admin CLI
    4. Insert school connection record into `school_connections` table
    5. Set `database_status = 'active'`
    6. Run `ApplySchoolMigrations` for the new school
    
    ## Migrating School from Shared to Dedicated DB
    1. Provision dedicated DB
    2. Set `database_status = 'migrating'` (stops new connections)
    3. Run pg_dump from shared DB → pg_restore to dedicated DB
    4. Update `database_name`, `database_host` in school_connections
    5. Set `database_status = 'active'`
    6. Verify queries on dedicated DB
    
    ## Backup Operations
    - Scheduled: Asynq cron at 02:00 daily
    - Retention: 14 newest backups per school
    - Manual: POST /api/v2/admin/schools/{id}/backup
    
    ## Restore Operations
    - WARNING: Restore overwrites the target database
    - Point-in-time: select backup by ID, initiate restore
    ```

    # SchoolCare v2 — Operations Runbooks
    
    ## Runbook 1: Tenant DB Connection Failures
    **Symptoms:** TENANT_DB_UNAVAILABLE errors, 503 responses for specific schools
    **Impact:** School cannot access data (others unaffected)
    **Steps:**
    1. Check school's `database_status` in core DB
    2. Verify ENCRYPTION_KEY is correct and unchanged
    3. Test DSN manually: `psql "host=... port=... user=... dbname=..."`
    4. Check ConnectionManager logs for eviction events
    5. If encrypted password corrupted: re-encrypt and update record
    6. Verify: school health check passes
    
    ## Runbook 2: Migration Failures
    **Symptoms:** database_status stuck on 'migrating', migration logs show ERROR
    **Steps:**
    1. Identify failed migration ID from logs
    2. Fix migration SQL if needed
    3. Apply manually or reset database_status to re-run
    4. Verify: CoreMigrationStatus shows all applied
    
    ## Runbook 3: Backup Failures
    **Symptoms:** ErrBackupInProgress for extended period, S3 upload errors
    **Steps:**
    1. Check inProgress map for stuck entries (restart clears them)
    2. Verify S3 credentials (IAM policy: s3:PutObject, s3:GetObject, s3:DeleteObject on bucket)
    3. Check disk space for temp files
    4. Check pg_dump timeout (default: 30 min)
    
    ## Runbook 4: Point-in-Time Recovery
    **Symptoms:** Data corruption, need to restore to specific time
    **Steps:**
    1. List available backups: ListBackups API
    2. Select backup by ID
    3. Initiate restore: RestoreTenantBackup API
    4. Run validation queries
    5. Verify school data correct
    ```
    
    Reference exact error codes from `backend/internal/errors/` and `backend/internal/backup/errors.go`.

    # SchoolCare v2 — Testing Guide
    
    ## Test Structure
    - Package-level tests in `*_test.go` files (same package)
    - `t.Parallel()` for independent tests
    - testify/assert for non-fatal, testify/require for fatal assertions
    - gin.CreateTestContext + httptest.NewRecorder for middleware/handler tests
    - go-sqlmock for DB-layer tests (when not testing SQL itself)
    - t.Skipf for environment-dependent tests
    
    ## Build Tags
    - `//go:build integration` for tests requiring Postgres or Redis
    - Unit tests (no build tag) must pass without external services
    
    ## Running Tests
    ```bash
    # Unit tests (no external services)
    make test-unit
    
    # Integration tests (requires Postgres + Redis at localhost:5432, localhost:6379)
    make test-integration
    
    # All tests
    make test-all
    
    # All tests (no Makefile)
    go test -count=1 -race -shuffle=on ./...
    ```
    
    ## Test Patterns (with examples)
    - Middleware: `schoolid_test.go`, `error_test.go` (gin.CreateTestContext)
    - Crypto: `encryption_test.go` (table-driven, roundtrip, tampering)
    - ConnectionManager: `connection_manager_test.go` (pre-populated cache, mock DBs)
    - Backup/Recovery: `service_test.go` in backup/restore (go-sqlmock for DB)
    - Queue handlers: `payloads_test.go` (serialization roundtrip)
    
    ## Writing Tests
    1. Follow existing patterns in the same package
    2. Use `//go:build integration` for external service dependencies
    3. Always use `t.Parallel()` for independent tests
    4. Always use `-race` flag when running tests
    5. Mock dependencies via interface extraction or package-level access
    
    ## CI Pipeline
    - `.github/workflows/ci.yml`: lint → unit-tests (30% coverage threshold) → integration-tests → build
    - Coverage threshold enforced: total >= 30%
    ```
