    **2a: Create** `docs/architecture/search-path-strategy.md`:

    Document the complete strategy in a structured architecture doc:

    ```markdown
    # Search Path Isolation Strategy

    ## Problem

    Academio uses schema-per-tenant isolation. Every query must target the correct
    tenant schema (`school_X`). PostgreSQL's `search_path` controls which schema
    unqualified table references resolve to — but setting `search_path` at the
    connection level is incompatible with PgBouncer transaction pooling because:

    1. PgBouncer multiplexes connections — a `SET search_path` on one connection
       can leak to the next user on the same connection.
    2. Even with `SET LOCAL`, the setting only survives inside a transaction —
       any query outside a transaction uses the default search_path.

    ## Strategy: GORM Plugin (SchemaTablePrefix)

    ## Decision: GORM Plugin over SET search_path

    | Approach | PgBouncer Compatible | Survives Connection Pool | Notes |
    |----------|---------------------|--------------------------|-------|
    | GORM SchemaTablePrefix plugin | ✅ Yes | ✅ Yes | Context-based, no connection state |
    | SET LOCAL search_path in tx | ✅ Yes (inside tx) | ❌ No (tx end = reset) | Correct for migrations |
    | SET search_path (connection-level) | ❌ No | ❌ No | Leaks across users |

    ## Code Paths

    ### API Queries (all GORM model operations)
    - **Mechanism**: SchemaTablePrefix GORM plugin
    - **Coverage**: All Create/Query/Update/Delete/Row/Raw callbacks
    - **File**: `backend/internal/database/tenant/schema_db.go`
    - **Correctness**: ✅ — schema prefix survives Statement.clone() via Set/Get

    ### Tenant Migrations
    - **Mechanism**: SET LOCAL search_path inside GORM transaction
    - **Coverage**: ApplySchoolMigrationsForSchema
    - **File**: `backend/internal/database/tenant/migration_service.go`
    - **Correctness**: ✅ — transaction boundary prevents connection leakage

    ### pg_dump / pg_restore
    - **Mechanism**: `--schema=` CLI flag
    - **Coverage**: BackupService.CreateTenantBackup, RestoreService.RestoreBackup
    - **Files**: `backend/internal/backup/service.go`, `backend/internal/restore/service.go`
    - **Correctness**: ✅ — explicit schema targeting via pg_dump flag

    ### CREATE SCHEMA
    - **Mechanism**: Explicit schema name in raw SQL
    - **Coverage**: ProvisionSchool(), CreateSchemaAndMigrate()
    - **Files**: `backend/internal/database/tenant/provisioning.go`, `migration_service.go`
    - **Correctness**: ✅ — schema name is constructed from school ID with regex validation
    ```
