<objective>
Build the Student Health Records module — a complete set of GORM models, CRUD API endpoints, and PDF export for school nurses to manage immunizations (NPI/EPI Nigerian schedule), allergy alerts, medication logs, and nurse visits.

**Purpose**: School nurses need a dedicated workflow to log and view student health data. The existing system has NO health-related fields on any model (UserInfo.Details is plain JSONB with no schema). This phase creates a purpose-built StudentHealth model family with proper FK relationships, tenant-scoped data access, and a PDF export for parent/guardian records.

**Output**: 5 new GORM models, 1 migration update, 4 module files (dto/handler/service/repository), 2 router wiring changes, and a working PDF export pipeline using the existing Gotenberg client.
</objective>

**Current state**: The system has NO health records module. UserInfo has a Details JSONB field but no structured health schema. The Gotenberg PDF client exists at `pkg/pdf/gotenberg.go` and is wired as `pdfGen` in setup.go. The existing `health` package (`internal/modules/health/`) is the system health-check module (livez/readyz) — the student health module uses a different package name (`studenthealth`) to avoid import collisions.

**Nigerian NPI/EPI schedule for reference**:
- Birth: BCG, OPV-0, HepB-0
- 6 weeks: OPV-1, Pentavalent-1, PCV-1, Rotavirus-1
- 10 weeks: OPV-2, Pentavalent-2, PCV-2, Rotavirus-2
- 14 weeks: OPV-3, Pentavalent-3, PCV-3, Rotavirus-3, IPD (inactivated polio)
- 6 months: Measles-1, Yellow Fever
- 9 months: Measles-2 (MCV2)
- 12-15 months: Measles-Rubella (MR), booster doses
- 18 months: DTP booster, OPV booster, MMR
- 5-7 years: DTaP/IPV booster, MMR booster
- 9-13 years: HPV (2 doses, 6 months apart), Tdap booster

**Models need `StudentID` FK back to `students.id` (not `users.id`)** — immunizations, allergies, medications, and visits are per-student, not per-user.

        VisionLeft *string `gorm:"type:varchar(10)" json:"vision_left,omitempty"`     // 6/6, 6/9, 6/12, etc.
