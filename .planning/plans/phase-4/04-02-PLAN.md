# 04-02: Stable Student Identifiers

## Goal
Ensure `admission_number` works as a cross-year stable key for student re-enrollment without data duplication.

## Work Done
- `admission_number` was already a unique column on `students` table — no new migration needed
- Re-enrollment endpoint: `POST /api/v2/students/:id/reactivate` — sets `deleted_at = NULL`, updates `session_id`
- Identifier generation: auto-generated on student creation, also settable via CSV import / bulk create
- Updated service methods to use `admission_number` for lookup during CSV import

## Files Changed
- `backend/internal/modules/user/service.go` — student reactivation
- `backend/internal/modules/user/repository.go` — `FindByAdmissionNumber`
