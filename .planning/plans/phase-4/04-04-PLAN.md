# 04-04: Student Promotion/Graduation + Frontend UI

## Goal
Auto-promote students to next class, graduate students (Alumni auto-create), and provide frontend UI for promotion workflow.

## Work Done
- **PromotionRecord model** — stores per-student JSON snapshot (scores, level, session, promotion status) for each promotion cycle
- **Alumni auto-create**: When a student graduates (promoted beyond highest level), an `Alumni` record is created inside the promotion transaction via log-and-continue (best-effort, non-blocking)
- `applyPromotions` accepts optional `sessionID` parameter to scope promotion to a specific session
- **Frontend**: Standalone `/promotion` route with Promote tab (select third-term session → preview cards with keep-behind toggle → confirm dialog) and History tab (completed third-term promotions)
- Sidebar link added under Academics group
- Group 22 migration creates `promotion_records` + FK constraint

## Files Changed
