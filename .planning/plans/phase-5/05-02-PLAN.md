# 05-02: Boundary Test Suite

## Objective
Unit and integration tests for WAEC A1-F9 grade boundary calculations at ±0.01 precision, and an integration test verifying frozen-grade read-only state via the ScoreGrid disabled prop.

## Tasks

### Task 1: WAEC boundary precision tests
1. Test `RoundToBasisPoints()` / `BasisPointsToFloat()` in `backend/pkg/mathutil/rounding.go`
   - Verify 75.00% → 7500 basis points
   - Verify boundary values: 74.995% rounds to 75.00% (boundary)
   - Verify 74.994% rounds to 74.99%
2. Test WAEC A1-F9 grade boundary comparison:
   - A1: ≥75.00%
   - B2: 70.00%–74.99%
   - B3: 65.00%–69.99%
   - C4: 60.00%–64.99%
   - C5: 55.00%–59.99%
   - C6: 50.00%–54.99%
   - D7: 45.00%–49.99%
   - E8: 40.00%–44.99%
   - F9: <40.00%
3. Test each boundary at exact threshold (e.g., 75.00% → A1, 74.99% → B2)
4. Test at ±0.01 precision to ensure no floating-point rounding errors

### Task 2: Frozen-grade read-only integration test
1. Write test that:
   a. Creates a session with status "completed"
   b. Calls the score modification endpoint
   c. Verifies the response is 403/error (grades frozen)
   d. Creates a session with status "active"
   e. Calls the same endpoint
   f. Verifies the response is 200 (grades modifiable)
2. Test the `canModifyScore()` helper returns error for completed sessions

### Task 3: ScoreGrid disabled state test (frontend)
1. Write a React component test for ScoreGrid:
   a. Render with `disabled={false}` → inputs are editable
   b. Render with `disabled={true}` → inputs are disabled, lock icons visible
   c. Verify `disabled` attribute on input elements
