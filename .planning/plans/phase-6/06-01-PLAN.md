# 06-01: CBA → Gradebook Score Push

## Objective
Connect CBA exam results to the academic gradebook. When an exam is auto-graded (SubmitExam) or all manual grades are complete (GradeAnswer), push the percentage score to the `Score` table so it appears in report cards and grade summaries.

## Tasks

### Task 1: Add `grade_item_id` FK to `CBAAssignment`
1. Add nullable `grade_item_id` column (uint) to `CBAAssignment` model — FK to `GradeItem`
2. Add migration in school migration file
3. Update `CreateCBAAssignmentRequest` DTO with optional `grade_item_id` / `grade_item_uuid`
4. Update `CreateAssignment` service to set `grade_item_id` if provided
5. Update `CBAAssignmentResponse` DTO to expose `grade_item_id`
6. Update `CreateAssignment` handler to accept the new field

### Task 2: Push score on auto-grade (SubmitExam)
1. In `SubmitExam` (cba/service.go), after auto-grading completes and session is submitted:
   - Resolve `grade_item_id` from the assignment
   - If present, calculate `percentage = (session.Score / session.TotalMarks) * 100`
   - Write to `Score` table: `{ student_id, grade_item_id, score: percentage, graded_by: "system", graded_at: now }`
   - Use upsert (ON CONFLICT student_id + grade_item_id → update score) for re-submissions

### Task 3: Push score on manual grade completion (GradeAnswer)
1. In `GradeAnswer`, after all answers for a session are graded:
   - Check `session.Score` recalculated after the grade
   - If `grade_item_id` set, push/update the percentage score to `Score` table
   - Use the same upsert pattern as Task 2

### Task 4: Score push helper
1. Extract score push logic into `pushScoreToGradebook(ctx, tenantDB, session, assignment) error`
2. Shared between SubmitExam and GradeAnswer
3. Log warning and continue if `grade_item_id` is nil (legacy assignment, scores remain standalone)
