# 06-02: CBA Entity Integration & Exam Config

## Objective
Replace free-text `Class`/`Subject` strings on `CBAAssignment` with proper foreign keys to `Level` and `Subject` models, and add exam configuration (schedule window, max attempts, passing threshold, question randomization). This enables cross-referencing CBA data with the academic module for reports and analytics.

## Tasks

### Task 1: Add `level_id` and `subject_id` FKs to `CBAAssignment`
1. Add nullable `level_id` (uint → `Level` model) and `subject_id` (uint → `Subject` model) columns
2. Add migration in school migration file
3. Keep existing `Class`/`Subject` string fields for display backward compatibility
4. Update `CreateCBAAssignmentRequest` DTO with optional `level_id`/`level_uuid`, `subject_id`/`subject_uuid`
5. Update `CreateAssignment` service to set FK fields if provided
6. Update `CBAAssignmentResponse` DTO to expose `level_id`, `subject_id`, plus resolved names via GORM Preload

### Task 2: Add `exam_config` JSONB column
1. Add `exam_config` column (JSONB, nullable) to `CBAAssignment` model
2. Defines a Go struct for the config:
   ```go
   type ExamConfig struct {
       ScheduleStart         *time.Time `json:"schedule_start,omitempty"`
       ScheduleEnd           *time.Time `json:"schedule_end,omitempty"`
       MaxAttempts           *int       `json:"max_attempts,omitempty"`  // nil = unlimited
       PassingThreshold      *float64   `json:"passing_threshold,omitempty"` // 0-100
       RandomizeQuestions    *bool      `json:"randomize_questions,omitempty"`
       ShowResultsImmediately *bool     `json:"show_results_immediately,omitempty"`
   }
   ```
3. Update `StartExam` to check schedule window (if ScheduleStart/End set)
4. Update `StartExam` to enforce MaxAttempts (count existing sessions for this student+assignment)
5. Update `SubmitExam` to use PassingThreshold for pass/fail determination (overrides hardcoded boundary)
6. Update `GetExamSession` to respect RandomizeQuestions when returning questions
7. Update `GetExamResults` to respect ShowResultsImmediately — if false, hide answers until teacher releases

### Task 3: Update CBA DTOs
1. `CreateCBAAssignmentRequest`: add `exam_config` field (JSON/RawMessage)
2. `CBAAssignmentResponse`: add `exam_config` field and resolved level/subject names
3. `ExamResultResponse`: add `passed` bool field based on PassingThreshold
