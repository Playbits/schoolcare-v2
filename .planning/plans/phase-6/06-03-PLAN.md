# 06-03: LMS Admin CRUD UI

## Objective
Build the teacher/admin interface for creating, editing, and deleting courses, modules, and lessons. Currently all CRUD is backend-only — teachers need a UI to manage course content without API calls.

## Tasks

### Task 1: Course admin dialog
1. Add "New Course" button on the course catalog page (`lms.tsx`)
2. Create course form dialog with react-hook-form + Zod:
   - Name, Code (optional slug), Description (textarea), Subject (select from useSubjects), Level (select from useClasses), Teacher (select from user list), Cover Image URL, Status (draft/published/archived)
3. Edit dialog — pre-filled same form, loads existing course data
4. Delete confirmation dialog — warns if enrollments exist (backend already blocks)
5. Wire to existing `useCreateCourse`, `useUpdateCourse`, `useDeleteCourse` hooks

### Task 2: Module management within course detail
1. On the course detail page (`lms.$courseId.tsx`), add an "Edit Modules" mode toggle
2. In edit mode, show the module list as a reorderable list (drag handle, simple up/down buttons)
3. Inline create: "Add Module" button → inline form (title, description)
4. Inline edit: click module title → edit in place
5. Delete with confirmation — warns if lessons exist (backend already blocks)
6. Wire to existing `useCreateModule`, `useUpdateModule`, `useDeleteModule` hooks

### Task 3: Lesson editor
1. Add lesson list within each module in edit mode
2. "Add Lesson" button → opens lesson editor dialog:
   - Title, Content Type (select: video/text/pdf/quiz), Duration (minutes)
   - Content URL (for video/pdf — file URL input)
   - Content Text (rich text editor for text content — use textarea with basic markdown or simple HTML)
   - Sort Order (number input)
   - Is Free (checkbox for preview)
3. Edit dialog — pre-filled same form
4. Delete confirmation
5. Wire to existing `useCreateLesson`, `useUpdateLesson`, `useDeleteLesson` hooks

### Task 4: Route gating and navigation
1. Ensure course catalog shows create/edit controls based on user role (admin/teacher)
2. Show "Manage Courses" link in sidebar for admin/teacher roles
3. Redirect non-admin/non-teacher away from edit mode
