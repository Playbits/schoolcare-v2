# 06-04: Student Progress Dashboard

## Objective
Give students a unified view of their learning: enrolled courses with progress, assignment scores, and CBA exam results. Depends on 06-01 for CBA ‚Üí gradebook integration so scores display alongside assignment grades.

## Tasks

### Task 1: "My Learning" section on LMS page
1. On the existing `lms.tsx` course catalog, add a "My Learning" tab or top section for students
2. Shows only courses the student is enrolled in (filter backend by enrollment)
3. Each course card shows:
   - Course name, cover image (small)
   - Progress bar (`enrollment.ProgressPct`)
   - "Resume" button ‚Üí links to course detail (opens at last uncompleted lesson)
4. Wire to `useEnrolledCourses` (new hook or filter on `useCourses` with `?enrolled=true`)

### Task 2: Progress stats cards
1. Add stats cards above the enrolled courses list:
   - "Courses Enrolled" (count)
   - "Lessons Completed" (sum of lesson_progress where completed=true)
   - "Assignments Pending" (count of assignments with no submission)
   - "Average Score" (from linked score/gradebook entries)
2. Fetch counts via existing hooks or computed from enrollment detail

### Task 3: Assignment scores on course detail
1. On `lms.$courseId.tsx`, add a "My Grades" section (student view) or tab
2. Shows assignment list with submission status and score
3. If graded: show score, feedback, graded date
4. If ungraded: show "Pending grading" badge
5. If not submitted: show "Not submitted" with due date
6. Wire to existing `useSubmissions` and `useAssignment` hooks
7. After 06-01: also show linked CBA exam scores alongside assignment scores

### Task 4: Backend ‚Äî enrolled courses endpoint or filter
1. Add `?enrolled=true` filter to `GET /lms/courses` handler
2. When `enrolled=true`, join through `course_enrollments` and return only courses where `student_id = currentUser.StudentID`
3. Include `ProgressPct` and enrollment status in response
