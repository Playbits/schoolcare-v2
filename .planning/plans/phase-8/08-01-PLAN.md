# 08-01: Widen AuditLogging Middleware + Response Enrichment

## Objective
Apply `AuditLogging` middleware to all authenticated route groups (not just `tenantAware`), fix `inferResourceType()` for parameterized routes, and add user/school name resolution to `AuditLogResponse`.

## Tasks

### Task 1: Widen AuditLogging middleware to all authGroup routes
1. In `router.go`, the `authGroup()` function creates an authenticated route group with JWTAuth, EnforceSchoolID, TenantResolution, TenantDBResolver. Currently, `AuditLogging` is only applied to the `tenantAware` group (line 143).
2. Add `sub.Use(middleware.AuditLogging(auditLogger))` inside the `authGroup()` function ŌĆö after all existing middleware. This will apply it to ALL routes using `authGroup()` (~36 route groups).
3. Remove the `tenantAware.Use(middleware.AuditLogging(auditLogger))` line to avoid double-logging.
4. Route groups NOT using `authGroup()` that still need audit logging:
   - **Admin group** (`/admin`) ŌĆö uses inline middleware, add `AuditLogging` there
   - **Schools group** (`/schools`) ŌĆö uses inline middleware, add `AuditLogging` there
   - **Auth protected routes** (`/auth` with JWT) ŌĆö uses inline middleware, add `AuditLogging` there
   - **Tenant-aware group** (`/tenants/config`) ŌĆö already has it (will be removed from here since authGroup covers it... actually `tenantAware` is NOT using `authGroup()` ŌĆö it's inline. So keep it on `tenantAware` but also add to `authGroup()`).

### Task 2: Fix inferResourceType for parameterized routes
1. The current `inferResourceType()` takes the last non-parameter segment. For paths like `/api/v2/users/:id` it returns `"users"` (correct). But for paths like `/api/v2/cba/exams/:id/results`, it would return `"results"` (correct).
2. Add a map or regex-based approach for known tricky patterns:
   - `/api/v2/user/student/:id` ŌåÆ `"student"`
   - `/api/v2/user/teacher/:id` ŌåÆ `"teacher"`
   - `/api/v2/user/staff/:id` ŌåÆ `"staff"`
   - `/api/v2/cba/answers/:id/grade` ŌåÆ `"cba_answer"`
   - Multi-segment resource paths should use the most specific resource name
3. Keep simple heuristic as fallback ŌĆö the audit entry is still useful even if resource_type is imperfect.

### Task 3: Add user/school name resolution to AuditLogResponse
1. Update `AuditRepository.FindBySchool()` to LEFT JOIN with `users` (public schema) and `schools` (public schema) tables to populate user name and school name.
2. Update `AuditLogResponse` DTO to add:
   - `UserName string` ŌĆö user's name (Firstname + Lastname from users table, or email as fallback)
   - `SchoolName string` ŌĆö school name from schools table
3. Update `toAuditLogResponse()` to populate the new fields.
4. Update `AuditRepository.FindByID()` similarly for the single-entry endpoint.
5. Since `users` table is in `public` schema (not tenant-scoped), query it via `r.db.WithContext(ctx).Table("users")` or core DB reference.
