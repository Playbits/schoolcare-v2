# 08-02: LogMutation Calls in Key Service Layers

## Objective
Add `LogMutation()` calls in 5 key service layers (User, School, Academic, Score, CBA) to capture `ResourceID`, `OldValues`, and `NewValues` for the most important mutations.

## Background
The `middleware.LogMutation()` helper exists but is never called anywhere. This plan adds calls in the 5 highest-impact modules. Each service needs access to the `AuditLogger` and request context (school_id, user_id, IP, user-agent, request_id).

## Architecture Decision
The `LogMutation()` function needs request context (user_id, school_id, IP, user-agent, request_id). Services don't have direct access to `*gin.Context`. Options:
1. Pass request context through service method parameters
2. Use `context.Context` to carry audit metadata (Gin context keys pattern)
3. Create a per-request audit context wrapper

**Decision**: Use middleware that stores audit context in `context.Context` (via the same keys already available — `GetUserID()`, `GetSchoolID()`), then services can call `middleware.LogMutation()` by passing the request context to service methods which is already the pattern (service methods receive `ctx context.Context`).

This works because:
- All handler methods already pass `c.Request.Context()` to services
- Gin stores user_id/school_id in context keys via existing middleware
- The `middleware` package already has `GetUserID()`, `GetSchoolID()`, `GetRequestID()` helpers
- Services can call these from the `ctx` (they check Gin context keys)

### Implementation Pattern
```go
// In service method:
func (s *UserService) CreateUser(ctx context.Context, req CreateUserRequest) (*models.User, error) {
    // ... create user logic ...
    
    // Audit log after creation
    newValues := &models.JSONMap{"email": req.Email, "role": req.Role}
    middleware.LogMutation(s.auditLogger,
        middleware.GetSchoolIDFromCtx(ctx),
        middleware.GetUserIDFromCtx(ctx),
        "create", "user", fmt.Sprintf("%d", user.ID),
        nil, newValues,
        middleware.GetClientIPFromCtx(ctx),
        middleware.GetUserAgentFromCtx(ctx),
        middleware.GetRequestIDFromCtx(ctx),
    )
    
    return user, nil
}
```

Services and handlers that already receive `ctx context.Context` need:
- Audit logger injected via constructor
- Context helpers to extract user/school/request metadata
