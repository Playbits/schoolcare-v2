# 08-03: Audit Archive / Retention System

## Objective
Create an `audit_logs_archive` table, a cron job to move logs older than 90 days from `audit_logs` to `audit_logs_archive`, and an admin query endpoint for archived logs.

## Tasks

### Task 1: Add archive table migration
1. Add `AuditLogArchive` model to `backend/internal/database/models/audit_log.go` (or same file):
   - Same schema as `AuditLog` (all columns identical)
   - Table name: `audit_logs_archive`
   - Lives in tenant schema (same `SchemaTablePrefix` behavior)
   - Auto-migrate in school migration file (Group 15 together with AuditLog)
2. Register in `backend/internal/database/models/schema.go`

### Task 2: Add archive service and repository
1. Create repository method in `audit/repository.go`:
   - `InsertArchiveBatch(ctx, entries []AuditLog) error` â€” batch insert to archive table
   - `DeleteOlderThan(ctx, schoolID uint, age time.Duration) (int64, error)` â€” delete from audit_logs
   - `FindArchived(ctx, schoolID uint, filter AuditFilter) ([]AuditLog, int64, error)` â€” query archive table
   - `GetArchivedByID(ctx, schoolID, id uint) (*AuditLog, error)` â€” single archive entry
2. Create archive job in `scheduler/jobs.go` or `backend/internal/scheduler/audit_archive_job.go`:
   - Enumerate all active schools
   - For each school: get tenant DB â†’ move logs older than 90d â†’ insert into archive â†’ delete from main
   - Run in a transaction per school: SELECT from main, INSERT into archive, DELETE from main
   - Log count of archived entries per school
   - Default schedule: daily at 1:00 AM (configurable via `CRON_AUDIT_ARCHIVE` env var)
3. Register archive job with `CronRunner` in setup.go

### Task 3: Add admin archive query endpoint
1. Add handler in `audit/handler.go`:
   - `ListArchived` â€” GET endpoint with same filtering as List but querying archive table
   - `GetArchived` â€” GET single entry from archive table

   - `GET /audit-logs/archive` â†’ `auditHandler.ListArchived`
   - `GET /audit-logs/archive/:id` â†’ `auditHandler.GetArchived`
3. Add service and repository methods as described in Task 2

### Task 4: Add swagger annotations for archive endpoints
- Add `@Summary`, `@Description`, `@Param`, `@Success`, `@Failure` annotations for the two new endpoints
