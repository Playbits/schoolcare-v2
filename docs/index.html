Quick Start

Prerequisites

  • Docker (Postgres :5432, Redis :6379)
  • Go 1.26
  • Node.js + Yarn 4+
  • Air (hot reload)

Setup Commands

# Backend
$ cd backend
$ cp .env.example .env
$ make db-init && make migrate && make seed
$ air

# Frontend (new terminal)
$ cd frontend
$ yarn install
$ yarn dev

Key Commands

  • make db-init DROP_TENANT=true Reset DB
  • make migrate && make seed Fresh migration
  • ./bin/server Start server
  • scripts/test_endpoint.sh Integration tests

Credentials

  • Super Admin: playbit / Password123!
  • Login: POST /api/v2/auth/login

Engineering Principles

Priority
Correctness > Security > Performance > Scalability > Simplicity > Maintainability
Log & Continue
Analytics/reports queries — log warning, continue. Best-effort; failing on transient blips is worse.
Return Errors
State mutations (create/update/delete) — return error. Silent failures corrupt status.
Batch Operations
Collect all errors, return one message — user fixes everything in one pass.
No Silent Discards
Every error must be handled. Never _ discard errors unless explicitly documented.
No Hardcoded Secrets
All credentials from env vars, validated at startup. Fail-fast on missing config.
Parameterized Queries
No fmt.Sprintf for SQL. Use GORM/pgx parameterized queries exclusively.
Unbounded Queries
Always cap list queries (default 100, max 1000). Use Offset + Limit.
Context Propagation
No context.Background() in request scope. Always propagate context.Context.

Database (150+ Models)

  • public schema: User, SchoolConnection
  • school_{id} schema: Student, Teacher, Level, Subject
  • Session, Curriculum, Assessment, Score, Result
  • CBA: Question, Paper, Assignment, ExamSession
  • LMS: Course, Module, Lesson, Enrollment, Submission
  • Finance: Account, Journal, Budget, Fee, Vendor
  • HR: Department, Staff, Leave, Payroll, Appraisal
  • Admissions: Intake, Application, Offer, Enrollment
  • Operations: Hostel, Transport, Inventory, Library
  • Engagement: Message, Notification, Alumni, Career

Multi-Tenancy

  • Schema-per-Tenant
    Each school gets its own schema school_{id} with isolated tables
  • SchemaTablePrefix Plugin
    GORM plugin auto-prepends schema name to all table references
  • Shared public Schema
    User accounts, school connections live in public
  • Tenant Resolution
    x-school-id header → DB connection → schema-scoped GORM instance

Authorization

  • JWT (Bearer Token)
    Access + refresh tokens with family-based rotation
  • School-scoped JWT
    EnforceSchoolID + TenantResolution middleware chain
  • Admin Only
    Super admin routes: impersonation, re-provision, schools list
  • 2FA / TOTP
    Optional two-factor authentication with recovery codes

Middleware Stack

Recovery RequestID Tracing Logger Security CORS RateLimit BodyLimit CSRF SchoolID JWTAuth TenantDB AuditLog

Job Queues (Asynq)

  • Critical
    Tenant provisioning, welcome emails, urgent notifications
  • Provisioning
    School schema creation, data seeding
  • Default
    Background tasks, batch operations
